# Ooho!語 リファレンス

## 概要
**設計方針**  
完全動的型のスクリプト言語。読みやすさを重視し自然言語風キーワードを採用する。ブロックは **スペース2つ** のインデントで表現する。値は必要に応じて自動変換され、直感的な記法を優先する。

**主要ルール（要点）**  
- **インデント**：スペース2つでブロックを表現する。  
- **動的型**：変数に型宣言は不要。演算や比較で自動変換。  
- **真偽値**：`0` が false、`0` 以外が true。比較・論理演算の結果は `0` または `1`。  
- **コメント**：行コメントは `Oh`、ブロックコメントは `Oo` ... `oO`。  
- **出力**：`oh` に続く `!` の個数で右側に必要な式の個数を指定する（例：`oh!` は 1 項、`oh!!` は 2 項）。

---

## 字句とコメント

### トークンの基本
- キーワードや記号は空白で区切ることを前提に設計する（ただしショートハンドは例外）。  
- **演算子（通常）** は前後にスペース必須（例：`a + b`）。  
- **ショートハンド**（`a++` 等）は一語トークンでスペース不要。  
- 文字列はダブルクォートで表記（例：`"Hello"`）。  
- 配列リテラルは `<` と `>` で囲む（例：`<1, 2, 3>`）。

### コメント
- **行コメント**：`Oh` を行内に置くと、その `Oh` 以降は行末までコメント扱い（`//` 相当）。  
```oh!! "値:" x  Oh ここは説明コメント```
- **ブロックコメント**：`Oo` で開始し `oO` で終了。複数行のコメントを囲む（`/* ... */` 相当）。  
```
Oo
複数行のコメント
oO
```
- **注意**：`//` はコメントトークンではない。

## 宣言と入出力

### 変数宣言
- `Ooho! x` — 1 変数宣言  
- `Ooho!! a b` — 2 変数宣言  
- `Ooho!!! p q r` — 3 変数宣言  
- `Ooho!` 系は宣言専用トークンとして字句解析で一義的に扱う。

### 入力
- `in(1)`：1 値を読み取り単一値を返す。  
- `in(n)`：n 個読み取り、配列 `<...>` を返す。  
- `in(x, y)`：x 列 y 行の 2D 配列を返す。  
- 区切りは空白・改行・タブを同等に扱う。

### 出力（感嘆符数による項数指定）
- 出力トークンは `oh` に続く `!` の個数で右側に必要な式の個数を決める。  
- `oh!` → 1 項出力（右に 1 式）  
- `oh!!` → 2 項出力（右に 2 式）  
- `oh!!!` → 3 項出力（右に 3 式）  
- 各式を左から評価し、**式間は単一スペース**で連結、**末尾に改行**を付ける。  
- **構文エラー**：必要な式数が満たされない場合は構文エラーとする。  
- 例:
```
oh! x
oh!! "合計:" total
oh!!! a b c
```

## データ型と値

### 動的型の原則
- 変数に型宣言は不要。値は整数、小数、文字列、配列などの内部表現を持つが、演算時に自動変換される。

### 数値
- 整数と小数を区別して保持。  
- 割り算は数学的に正しい小数を返す（例：`5 / 2 → 2.5`）。

### 文字列
- ダブルクォートで表記。  
- 文字列結合は `_`（アンダースコア）を使用し、**前後にスペース必須**（例：`a _ b`）。`_` は式内で先に評価される。

### 配列
- リテラル：`<1, 2, 3>`。  
- **1-indexed**：最初の要素は `a[1]`。  
- ジャグ配列（行ごとに長さが異なる）を許可。空配列は `< >`。  
- `len(a)` は最上位配列の長さ（行数）。`len(a[i])` は i 行目の要素数。  
- インデックスは整数化される（例：`"2"` → `2`）。変換不可なら実装での扱い（エラーまたは `null`）を適用。

### 真偽
- 条件評価は `0` を false、それ以外を true とする。  
- 比較・論理演算の結果は `0` または `1`。

## 演算子と式

### スペース規則
- **通常演算子**（`+ - * /` 等）は前後にスペース必須。  
- **ショートハンド**（`a++` 等）はスペース不要。  
- **文字列結合 `_`** は前後にスペース必須。

### 算術演算
- 加算：`a + b`（数値加算）  
- 減算：`a - b`  
- 乗算：`a * b`  
- 剰余：`a % b`  

### 割り算ファミリー
| 記法 | 意味 |
|------|------|
| `a / b` | 通常割り算（小数可） |
| `a /- b` | 切り捨て（floor） |
| `a /+ b` | 切り上げ（ceil） |
| `a /~ b` | 四捨五入（round） |

### ショートハンド単項
- `a++`：`a = a + 1`  
- `a--`：`a = a - 1`  
- `a**`：`a = a * 2`（倍化）  
- `a//`：`a = a / 2`（半分化）

### 比較と論理
- 比較：`==`, `!=`, `>`, `<`, `>=`, `<=`（結果は `0`/`1`）  
- 論理：`&&`, `||`, `!`（結果は `0`/`1`）

### 優先順位
- 標準的な算術優先順位を採用。`_`（文字列結合）は式内で先に評価される。

### 例
```
x = "10"
y = 3

oh!! x + y      # 出力: 13
oh!! x _ y      # 出力: 103
oh!! 5 / 2      # 出力: 2.5
oh!! 5 /- 2     # 出力: 2
x++
```

## 制御構造

### if 系
```
oh? 条件
<ブロック>
oh oh? 条件
<ブロック>
oh
<ブロック>
```
- `oh?` が if、`oh oh?` が else-if、`oh` が else。  
- 条件は `0` → false、それ以外 → true。ブロックはスペース2つでインデント。

### while 相当
```
Ooho 条件
<ブロック>
```
- `Ooho` は while 相当。条件が真の間繰り返す。

### for（ho 文）
- 記法：`ho i range(a, b)`  
- 意味：`i` は自動生成され整数化される。`a` から `b` まで **終了値を含む**。  
- 逆順：`a > b` の場合は自動でステップ `-1`。  
- ステップ指定：`ho i range(a, b, s)`（負のステップ可）。  
- ループ変数はループ内ローカルで、ループ終了後は参照不可。

### 例
```
ho i range(1, n)
oh? a[i] / 2 == a[i] /~ 2
oh!! i _ "は偶数"
oh
oh!! i _ "は奇数"
```

## 配列操作と len

### len の挙動
- `len(a)`：最上位配列の長さ（行数）を返す。  
- `len(a[i])`：i 行目の要素数を返す。  
- 空配列は `0`。

### インデックスと範囲
- インデックスは 1-indexed。  
- 範囲外アクセスは実装でエラー、`null`、または例外のいずれかを選択して扱う。

### ジャグ配列例
```
a = <
<1, 2>,
<3, 4, 5>,
<6>

len(a)    # 3
len(a[1]) # 2
len(a[2]) # 3
len(a[3]) # 1
```

---

## エラー処理と互換性
- **構文エラー**：トークンの不足（例：`oh!!` の後に式が 2 個未満）やスペース規則違反は構文エラー。  
- **ランタイムエラー**：インデックス範囲外、数値変換失敗（実装で例外か `null` を返す）など。  
- **互換性**：`oh` 系の感嘆符数は言語仕様で上限を設けることができる（例：最大 5 個）。上限は実装で決定するが、仕様上は固定上限を推奨。

---

## 字句解析・パーサ実装メモ
- **トークン化**：`oh` + `!` 連続は `OH_BANG(n)` トークンとして扱う（`n` は `!` の個数）。  
- **コメント優先度**：`Oo` / `oO` / `Oh` は字句解析で優先的に処理する。  
- **スペースルール**：通常演算子の前後スペース必須ルールを厳格にチェックする。  
- **式パース**：`OH_BANG(n)` を見つけたら直後に n 個の式をパースする。式の終端は次の式の先頭や行末で判定する。  
- **1-indexed 内部表現**：内部で 1-indexed を扱うか、アクセス時に変換レイヤを用意する。  
- **ジャグ配列**：可変長の行を自然に扱える内部表現（配列の配列）を推奨。

## 付録：設計上の決定事項（要約）
- 出力は `oh` に続く `!` の個数で右側の式数を厳密に指定する。  
- 宣言は `Ooho!` 系で行い、`oh` 系と明確に分離する。  
- コメントは `Oh`（行）と `Oo`/`oO`（ブロック）で扱う。  
- 配列は 1-indexed、ジャグ配列を許可、`len` は最上位長を返す。  
- 真偽は `0`/`1`、比較・論理は `0`/`1` を返す。  
- 演算子の前後スペース規則を厳格にすることで曖昧性を減らす。
