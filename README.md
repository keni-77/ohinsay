# Ooho!語 リファレンス

## 概要
**設計方針**  
Ooho!の教えをいかした完全動的型のスクリプト言語。値は必要に応じて自動変換され、直感的な記法を優先。

## 字句とコメント

### トークン
- 通常演算子は前後スペース必須  
- ショートハンドはスペース不要

### コメント
- **行コメント**：`Oh` を行内に置くと、その `Oh` 以降は行末までコメント扱い（`//` 相当）。  
- **ブロックコメント**：`Oo` で開始し `oO` で終了。複数行のコメントを囲む（`/* ... */` 相当）。  
```
Oh (ここがコメント)
Oo
(複数行のコメント)
oO
```
- **注意**： `//` はコメントトークンではない。

## データ型と値

### 動的型の原則
- 変数に型宣言は不要。値は整数、小数、文字列、配列などの内部表現を持つが、演算時に自動変換される。

### 数値
- 整数と小数を区別して保持。  
- 文字列型になっている整数は自動変換（例：`"5" + 2` → `7`）。

### 文字列
- ダブルクォートで表記。（例：`"Ooho!"`）  
- 文字列結合は `_`（アンダースコア）を使用し、**前後にスペース必須**（例：`"和：" _ sum`）。`_` は式内で先に評価される。

### 配列
- リテラル：`<1, 2, 3>`。  
- **1-indexed**：最初の要素は `a[1]`。  
- ジャグ配列（行ごとに長さが異なる）を許可。
- 空配列は `<>`。  
- `len(a)` は最上位配列の長さ（行数）。`len(a[i])` はi行目の要素数。空配列なら `0`  
- インデックスは整数化される（例：`"2"` → `2`）。変換不可なら実装での扱い（エラーまたは `null`）を適用。

### 真偽
- 条件評価は `0` を false、それ以外を true とする。  
- 比較・論理演算の結果は `0` または `1`。

## 宣言と入出力

### 変数宣言
- `oh x` — 単数変数宣言
- `oh a b` — 複数変数宣言  
- `oh` は宣言専用トークンとして字句解析で一義的に扱う。
- 変数宣言時に値を代入することは出来ない。

### 入力
- `a = in(1)` ：1つの値を読み取り単一値を返す。  
- `b = in(n)` ：n個読み取り、配列 `<...>` を返す。  
- `c = in(x, y)` ：x列y行の2次元配列を返す。**注意**：x行y列ではない。  
- 区切りは空白・改行・タブを同等に扱う。

### 出力
- `say a`：単数出力  
- `say a b c`：複数出力  
- 各式を左から評価し、**式間は単一スペース**で連結、**改行**は"\n"。  
```
say a b c "\n"
say "合計：" total
```

## 演算子と式

### スペース規則
- **通常演算子**（`+` `-` `*=` `%=` 等）は前後にスペース必須。  
- **ショートハンド**（`a++` 等）はスペース不要。  
- **論理演算子**の否定記号 `!` の後はスペース不要。  
- **文字列結合** `_` は前後にスペース必須。  
- **単行マイナス** `-a`等はスペースなし。

### 基本算術演算
- 加算：`a + b`  
- 減算：`a - b`  
- 乗算：`a * b`   
- 剰余：`a % b`

### 除算
| 記法 | 意味 |
|------|------|
| `a / b` | 通常の割り算（小数が返る） |
| `a /- b` | 切り捨て（floor） |
| `a /+ b` | 切り上げ（ceil） |
| `a /~ b` | 四捨五入（round） |

### 複合代入
- 加算：`a += b`  
- 減算：`a -= b`  
- 乗算：`a *= b`  
- 剰余：`a %= b`  
- 文字列結合：`a _= b`

### 除算
| 記法 | 意味 |
|------|------|
| `a /= b` | 通常の割り算（小数に変化） |
| `a /-= b` | 切り捨て（floor） |
| `a /+= b` | 切り上げ（ceil） |
| `a /~= b` | 四捨五入（round） |

### ショートハンド単項
- `a++`：`a = a + 1`  
- `a--`：`a = a - 1`  
- `a**`：`a = a * 2`（倍化）  
- `a//`：`a = a / 2`（半分化）

### 比較と論理
- 比較：`==`, `!=`, `>`, `<`, `>=`, `<=`（結果は `0` または `1`）  
- 論理：`&&`, `||`, `!`（結果は `0` または `1`）

### 優先順位
- 標準的な算術優先順位を採用。`_`（文字列結合）は式内で先に評価される。

### 例
```
x = "10"
y = 3

say x + y   Oh 出力: 13
say x _ y   Oh 出力: 103
say x y    Oh 出力: 103
x++
say x / 2   Oh 出力: 5.5
say 5 /- 2  Oh 出力: 2
```

## 制御構造

### oh?文
- `oh?` は条件式が真であるとき実行する（`if` 相当）。  
- `oono` は1つ上の `oh?` または `oono oh?` が実行されなかったときに実行する（`else` 相当）。  
- 条件は `0` → 真、それ以外 → 偽。  
- ブロックはスペース2つでインデント。
```
oh? a > b
  say a ">" b
oono oh? a < b
  say a "<" b
oono
  say a "=" b
```

### Ooho文
- `Ooho` は `while` 相当。条件が真の間繰り返す。
```
a = 1
Ooho a <= 5
  say a
  a++
```

### ho文
- 記法：`ho i range(a, b)`  
- 意味：`i` は自動生成され整数化される。`a` から `b` まで**終了値を含む**。  
- 逆順：`a > b` の場合は自動でステップ `-1`。  
- ステップ指定： `ho i range(a, b, s)` （負のステップ可）。  
- ループ変数はループ内ローカルで、ループ終了後は参照不可。
```
n = 5
ho i range(1, n)    Oh i: 1, 2, 3, 4, 5
ho i range(1, 5, 2)    Oh i: 1, 3, 5
ho i range(3, 16, 4)    Oh i: 3, 7, 11, 15
```

## 配列操作と len

### len の挙動
- `len(a)`：最上位配列の長さ（行数）を返す。  
- `len(a[i])`：i 行目の要素数を返す。  
- 空配列は `0`。

### インデックスと範囲
- インデックスは 1-indexed。  
- 範囲外アクセスは実装でエラー、`null`、または例外のいずれかを選択して扱う。

### ジャグ配列
```
a = <
<1, 2>,
<3, 4, 5>,
<6>
>
say len(a)    Oh 3
say len(a[1])    Oh 2
say len(a[2])    Oh 3
say len(a[3])    Oh 1
```

## エラー処理と互換性

## エラー

### 構文エラー（仕様で定義）
- スペース規則違反  
- 未宣言変数の使用  
- 括弧の不一致  
- `say` の後に式が無い  
- 配列リテラルの構文ミス  
- キーワードの誤用
---

### ランタイムエラー（仕様で定義）
- 型変換失敗
```
a = "abc"
b = a + 3
```
- 0での除算
```
say 5 /- 0
```
- 無効な range（ステップ 0）
```
ho i range(1, 2, 0)
```
- 範囲外アクセス
```
a = <1, 2>
say a[3]
```
